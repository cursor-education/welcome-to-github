				3.What is differenct between HTTP version 1.0 and 1.1?
У HTTP 1.0 кожен запит створює нове з'єднання TCP, в той час як в HTTP 1.1 браузер створює одне з'єднання TCP і відправляє багато запитів через це.
HTTP 1.1 має необхідний заголовок хоста у специфікації. 
HTTP 1.0 офіційно не вимагає заголовок хоста, але це не завадить додати його тому, що багато додатків (проксі) очікують побачити заголовок хоста незалежно від версії протоколу.
HTTP 1.1 також дозволяє мати постійні з'єднання, що дозволяє здійснювати більше одного запиту/відповіді на тому ж з'єднанні HTTP. 
У HTTP 1.0 доводиться відкрати нове з'єднання для кожної пари запит/відповідь. І після кожної відповіді сеанс буде закрито. Це призвело до великих проблем з ефективностю та повільного старту TCP.
НTTP / 1.1 вводить метод OPTIONS. Клієнт HTTP може використовувати цей метод для визначення здатності сервера HTTP. Хоча це не дуже популярно сьогодні, велика частина цієї інформації передається по відповіді сервера.
кешування:
HTTP 1.0 була підтримка кешування через заголовок: If-Modified-Since. 
HTTP 1.1 розширює підтримку кешування використовуючи те, що називається "мітка об'єкта". Якщо 2 ресурси однакові, то вони будуть мати ті ж теги сутностей. 
HTTP 1.1 також додає If-Unmodified-С, If-Match, If-None-Match умовних заголовків. 
Є також подальші доповнення, які стосуються кешування Cache Control заголовка.
Існує новий код повернення в HTTP / 1.1 100 Continue. Це повинно застережити клієнта від відправки великого запиту, коли цей клієнт не впевнений що, сервер може обробити запит, або уповноважений опрацювати запит. У цьому випадку клієнт відправляє тільки заголовки і сервер буде повідомляти клієнта.
HTTP 1.1 в цілому дає більше свободу дій( модернізує обмеження HTTP 1.0)
HTTP 1.0 розпізнає 16-значні коди.
HTTP 1.1 розпізнавати 24-значні коди.
HTTP 1.1 є безпечним, так як він використовує контрольне ім'я користувача, пароль і одним часове значення.

При використанні функції "сортування", можна відсортувати результати за одним із з значень колонки: статусом, типом, розміром, та іншими параметрами шляхом натискання на назву колонки. 
При використанні функції "пошук", можна значно спростити собі життя. При вводі тексту в поле, можна з великого нагромадження інформації швидко знайти потрібний фрагмент коду. 
Функція "фільтр" дозволяє фільтрувати отриману інформацію шляхом відображеня лише певних категорій. Також існують ключові слова, які фільтрують дані за типом домену, розміром файла, наявністю відповіді від сервера та іншими параметрами.
